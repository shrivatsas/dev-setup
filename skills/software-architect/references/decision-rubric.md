# Decision Rubric

Use this rubric to make recommendations sharp and defensible.

## Rank candidate approaches by:

1. Fit to the stated outcome
2. Complexity added
3. Operational burden
4. Delivery speed
5. Cost to build and run
6. Security and compliance fit
7. Ease of future change
8. Team capability fit

## Recommendation rules

- Prefer the simplest solution that satisfies the outcome and constraints.
- Reject options that require major operational maturity the team does not have.
- Call out hidden costs: migrations, observability, on-call, data quality, vendor lock-in.
- If two options are close, recommend the one with lower long-term complexity.
- If uncertainty is high, say what decision would reduce it fastest.
