# Research Deep

Use this skill when the user wants the deep research phase after the outline and field framework are ready.

## Response Style

- Lead with the answer or the strongest finding.
- Keep prose short and direct.
- Prefer concrete evidence, dates, names, and sources over abstract commentary.
- Separate facts, inferences, and assumptions.
- Avoid hedging unless uncertainty is real and important.

## Trigger

`/research-deep`

## Workflow

1. Read the current outline, items, fields, and time range.
2. Run deeper source collection for each item and each uncertain field.
3. Verify claims against sources and note conflicts explicitly.
4. Produce a synthesis that is structured, evidence-backed, and easy to scan.
5. Call out unresolved gaps instead of smoothing them over.

## Output Shape

- Executive summary
- Key findings
- Evidence table or source-backed notes
- Open questions / gaps
- Recommended next step
